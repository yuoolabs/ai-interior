# 微页面智能搭建助手 Demo

## 当前能力
- 输入需求（文字 + 行业/目标/风格 + 可选参考图）
- 上传参考图后，会自动判断配色、风格、页面目标和建议模块
- 先走规划器，生成页面方案
- 生成完成后可点击“开始自动搭建到后台”，会在当前窗口新开一个标签页进入微页面后台开始执行
- 自动搭建固定从微页面列表页起跑，再点击“新建页面”进入编辑器，避免直接跳去应用首页
- 如果系统把页面跳回应用首页，自动流程会直接停下，不再在两个页面之间反复跳转
- 如果页面提示 Chrome 没打开外部控制开关，需要先在 Chrome 顶部菜单打开：`视图 -> 开发者 -> Allow JavaScript from Apple Events`
- 再走执行器，输出 Chrome DevTools MCP 动作清单
- 内置一条可直接复用的 4 步真实执行链
- 已补齐图片、优惠券、商品三类真实补内容动作
- 已补上富文本默认文案动作和链接导航补项动作模板
- 已在真实后台跑通一条卖货页整链：建页、加组件、补核心内容、补文案、补导航、保存回列表
- 最后走校验器，给出限制提醒和替代说明

## 代码结构
- `demo/core/planner.js`：需求解析、模板选择、组件映射
- `demo/core/executor.js`：浏览器执行清单生成
- `demo/core/bootstrap-run.js`：首条真实执行链（4 步）
- `demo/core/validator.js`：校验与差异说明
- `demo/rules.js`：真实后台组件规则

## 如何运行
```bash
cd demo
node server.js
```
打开 `http://127.0.0.1:8001`

如果只是用 `python3 -m http.server` 打开静态页面，`开始自动搭建到后台` 不会真正执行，因为它需要本地服务来发起后台操作。

## 当前边界
- 先做外挂式方案和动作清单，不直接改内部系统
- 组件范围固定为已采集到的 26 个真实组件
- 当前已验证能补券、补商品、补图片、补富文本文案、补一个链接导航项，并把导航项连到现有页面
- 当前仍以现有素材、现有商品、现有页面为前提
- 参考图识别是 Demo 级能力，按画面颜色、分区和文件名线索做判断，不追求 100% 识别
