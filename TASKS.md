# 插件 + Chrome DevTools MCP 落地任务

## 完成标准
- 能把一句话需求转成标准页面方案
- 能把页面方案映射到真实后台组件
- 能产出可执行的浏览器动作清单
- 能在执行后给出校验和降级说明

## 第一阶段：先跑通最小闭环
1. 需求解析
   产出：页面目标、行业、风格、模块列表
   验收：卖货页、活动页、会员页都能稳定识别
2. 模板规划
   产出：固定结构模板
   验收：每种页面至少有一套可复用结构
3. 组件映射
   产出：模块 -> 真实组件映射
   验收：全部命中 26 个真实组件范围，不出现虚构组件
4. 浏览器执行脚本
   产出：按步骤的 MCP 动作清单
   验收：包含新建页、加组件、保存、预览
5. 结果校验
   产出：结构检查、限制检查、替代说明
   验收：能指出缺失模块和降级项

## 第二阶段：接真实模拟操作
1. 固定登录态进入装修页
2. 点击新建页面
3. 按方案顺序添加组件
4. 填默认内容
5. 保存并打开预览

## 第三阶段：补真实业务数据
1. 商品数据填充
2. 优惠券和活动数据填充
3. 图片和跳转配置
4. 发布前校验
